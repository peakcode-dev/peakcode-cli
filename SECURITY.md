# Security Policy

## Reporting a Vulnerability

**DO NOT open a public issue!**
Security vulnerabilities must **never** be reported through public GitHub/GitLab issues,
discussions, or any other publicly visible channel. Public disclosure of a vulnerability
before it is patched puts all users at risk.

Please contact us privately using one of the methods below. We prefer **Signal** for
secure, encrypted communication.
